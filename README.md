> **ReadDecay** is a local-first reading list manager with a twist вҖ” every link you save has a **freshness score** that exponentially decays over time. The longer you wait, the staler it gets. Links you haven't read sort to the top, glowing red, demanding your attention. Read them, snooze them, or watch them fade.

## вңҰ The Concept

Most read-later apps treat your reading list like a warehouse вҖ” links pile up forever, sorted by date, never asking you to act. **ReadDecay treats your reading list like a perishable.** Every link has a shelf life.

The core mechanic is **exponential decay**:

```
freshness(t) = 0.5 ^ (effectiveAge / halfLife)
```

A link saved today scores `1.0`. After one half-life (default: 7 days), it scores `0.5`. After two, `0.25`. The inbox sorts by ascending freshness вҖ” **the most stale links rise to the top**, creating urgency through visual pressure. A color-coded bar shifts from `green вҶ’ yellow вҶ’ red` as time passes.

It's gamified procrastination therapy.

## вңҰ Feature Set

### рҹ“Ҙ Inbox
- **Automatic metadata fetching** вҖ” saves a URL and immediately fetches the page title and favicon in the background (with Open Graph support and Google S2 favicon fallback)
- **Freshness-sorted list** вҖ” most stale links always surface at the top
- **Per-card freshness bar** вҖ” animated, color-coded `green вҶ’ yellow вҶ’ red`
- **Numeric score badge** вҖ” exact freshness value (0.00 вҖ“ 1.00) on each card
- **Age label** вҖ” human-readable ("just now", "3 days ago", "2 wks ago")
- **Domain + favicon** вҖ” extracted from URL, lettermark fallback if favicon fails
- **Tag chips** вҖ” inline tag display on each card (up to 3 visible)
- **Half-life info bar** вҖ” shows current decay rate at the top of the list
- **Link count badge** вҖ” live count of inbox items in the app bar

### рҹ‘Ҷ Gestures
| Gesture | Action |
|---|---|
| **Tap** | Open link in external browser |
| **Swipe right вҶ’** | Mark as read (green confirmation + Undo) |
| **Swipe left вҶҗ** | Archive link (blue confirmation + Undo) |
| **Long press** | Open snooze sheet |

### рҹҳҙ Snooze
- **1 day** вҖ” back tomorrow
- **3 days** вҖ” back in 3 days
- **1 week** вҖ” back next week
- Decay **pauses** for the entire snooze duration (effective age freezes)
- Snoozed items show a `вҸё Snoozed until Jun 18` badge on the card
- Snooze history accumulates вҖ” multiple snoozes stack correctly

### вһ• Add Links
- **FAB / "+" button** in the app bar
- Paste any URL manually into the add sheet
- **Paste from clipboard** button вҖ” one tap
- Validates and auto-normalizes URLs (adds `https://` if missing)
- Shows animated loading state while fetching metadata
- **Share sheet** (Android) вҖ” share any URL from Chrome, Safari, Twitter, etc. directly into ReadDecay

### рҹ“Ӯ Archive
- All read and archived links in one searchable screen
- **Full-text search** вҖ” filters by title, domain, and tags live as you type
- **Status filter chips** вҖ” All / Read / Archived
- **Tag filter chips** вҖ” dynamically generated from all tags in your library
- Result count indicator
- Tap any archived link to reopen in browser
- Status dot (рҹҹў Read В· рҹ”ө Archived) on each card

### вҡҷпёҸ Settings
| Setting | Default | Range |
|---|---|---|
| Half-life | 7 days | 1 вҖ“ 30 days |
| Notification threshold | 25% freshness | 5% вҖ“ 90% |
| Daily notifications | Enabled | On / Off |
| Dark mode | Dark | Dark / Light |

### рҹ”” Notifications
- **Daily at 9 AM** local time вҖ” checks for stale links below threshold
- Smart batching: "3 saved links are getting stale вҖ” time to catch up!"
- Permission requested at runtime (Android 13+, iOS)
- Persists across device reboots (BOOT_COMPLETED receiver)
- Cancels automatically when no stale links exist

### Freshness Decay Model

```dart
// Effective age accounts for total time spent snoozed
score = pow(0.5, effectiveAgeDays / halfLifeDays)

// If currently snoozed, remaining snooze time is subtracted
// from effective age вҖ” decay is completely paused
```

| Score | Label | Color | Bar |
|---|---|---|---|
| `> 0.80` | Fresh | рҹҹў Green | `в–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–‘в–‘` |

## вңҰ Android Share Sheet Setup

ReadDecay registers itself as a handler for `text/plain` share intents. Once installed, you'll see **"ReadDecay"** appear in the Android share sheet from any app (Chrome, Firefox, Twitter, etc.).

The app handles two scenarios:
1. **App is open** вҖ” the shared URL is saved immediately
2. **App is closed** вҖ” the URL is captured when the app launches

---

## вңҰ Notification Setup

On first launch, ReadDecay requests notification permission (Android 13+ / iOS). If granted:
- A daily check runs at **9:00 AM local time**
- If any inbox links are below the freshness threshold, a notification is sent
- The notification survives device reboots via the `BOOT_COMPLETED` receiver

To adjust thresholds, go to **Settings вҶ’ Notifications**.

## вңҰ Development Notes

### Re-running Code Generation

Any change to `lib/data/database.dart` requires a codegen re-run:

```bash
dart run build_runner build --delete-conflicting-outputs
```

### Riverpod Architecture

All business logic lives in **providers**, not in widgets:

```
UI вҶ’ ref.watch(provider)     # read reactive state
UI вҶ’ ref.read(notifier)      # trigger actions
```

`LinkActionsNotifier` is the single point of truth for all mutations вҖ” insert, read, archive, snooze, delete, and settings updates all go through it.

### Freshness is Never Stored

The freshness score is **computed at render time**, not persisted. This means:
- No migration headaches if the formula changes
- Changing the half-life in settings retroactively re-scores all links
- Snooze is stored as `snoozed_until` + cumulative `snoozed_seconds`
