## вңҰ The Concept

Most read-later apps treat your reading list like an infinite warehouse вҖ” links pile up forever, sorted by date, never asking you to act. **LinkShelf treats your reading list like a perishable.** Every link has a shelf life.

The core mechanic is **exponential decay**:

```
freshness(t) = 0.5 ^ (effectiveAge / halfLife)
```

A link saved today scores `1.0`. After one half-life (default: 7 days), it scores `0.5`. After two, `0.25`. The inbox sorts by ascending freshness вҖ” **the most stale links rise to the top**, creating urgency through visual pressure. A color-coded bar shifts from `green вҶ’ yellow вҶ’ orange вҶ’ red` as time passes.

It's gamified procrastination therapy.

## вңҰ Implemented Features

### рҹ–ҘпёҸ Desktop Widescreen Layout (Widescreen Mode)
- **Left Sidebar Navigation (`_DesktopSidebar`)** вҖ” When window width is >600px, LinkShelf automatically transforms from mobile bottom nav to a widescreen layout featuring a persistent left sidebar with bold branding and interactive highlights.
- **Widescreen Centered Dialogs** вҖ” Pushes `AddLinkSheet` inside a beautifully bounded, rounded widescreen Dialog (450px) instead of sliding up from the bottom of the screen.
- **Dynamic Columns Grid Math** вҖ” Calculates grid dimensions dynamically on the Folders screen to lay out folders appropriately depending on display size.
- **Dynamic Spacings** вҖ” Adjusts bottom scroll list spacing to automatically shrink from mobile pill padding (100вҖ“200px) down to standard desktop padding (40px) when the bottom floating navbar collapses.
- **Fluid Layout Resizing** вҖ” Narrowing the desktop app below 600px automatically transitions the UI seamlessly back into the compact phone layout.

### рҹҢҗ Browser Extension Popup (Extension Mode)
- **Manifest V3 Compliant** вҖ” Extension architecture compiles to comply with Manifest V3 and strict Content Security Policies (CSP).
- **Tab Auto-Detection** вҖ” Harnesses a custom JS Interop service (`chrome.tabs.query`) to parse the active browser tab's URL automatically and populate the ingestion field instantly when the popup opens.
- **Fixed Extension Viewport** вҖ” Configured viewport dimensions (400x600px) to render the app's phone layout natively inside Chrome and Safari toolbar dropdown panels.

### рҹ“Ҙ Inbox
- **Automatic metadata fetching** вҖ” Saves a URL and immediately fetches the page title, description, cover image, reading time estimation, and favicon in the background (using Open Graph parser + Google S2 favicon fallback).
- **Freshness-sorted list** вҖ” The most stale links always surface at the top to highlight reading priority.
- **Per-card freshness bar** вҖ” Animated, color-coded status bar (`green вҶ’ yellow вҶ’ orange вҶ’ red`).
- **Numeric score badge** вҖ” Displays exact freshness value (0.00 вҖ“ 1.00) directly on each card.
- **Age label** вҖ” Human-readable age representation ("just now", "3 days ago", "2 wks ago").
- **Domain + favicon** вҖ” Extracted from URL, with lettermark fallback if favicon fails.
- **Tag chips** вҖ” Inline tag display on each card (up to 3 visible tags).
- **Half-life info bar** вҖ” Shows current global half-life decay rate at the top of the list.
- **Link count badge** вҖ” Live count of inbox items in the app bar.
- **Full-text search** вҖ” Filters by title, domain, and tags live as you type.
- **Dead link indicator** вҖ” Warns you about dead or unreachable pages with a `вҳ пёҸ DEAD LINK` banner.

### рҹ“Ӯ Collections & Folder System

- **Tactile Card Stack Design** вҖ” Visual grid layout styling each folder as a physical directory tab with layered nested cards.
- **Spring Scale Animations** вҖ” Grid items feature dynamic haptic feedback and custom spring micro-scaling on tap.
- **Dynamic Folders Dashboard** вҖ” Integrated overview metrics displaying total folder count, percentage of links organized, and general decay average.

### Freshness Decay Model

```dart
// Effective age accounts for total time spent snoozed
score = pow(0.5, effectiveAgeDays / halfLifeDays)

// If currently snoozed, remaining snooze time is subtracted
// from effective age вҖ” decay is completely paused.
```

| Score | Label | Color | Bar |
|---|---|---|---|
| `> 0.80` | Fresh | рҹҹў Green | `в–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–‘в–‘` |
| `0.50 вҖ“ 0.80` | Fading | рҹҹЎ Yellow | `в–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–‘в–‘в–‘в–‘в–‘` |
| `0.25 вҖ“ 0.50` | Stale | рҹҹ  Orange | `в–Ҳв–Ҳв–Ҳв–‘в–‘в–‘в–‘в–‘в–‘в–‘` |
| `< 0.25` | Critical | рҹ”ҙ Red | `в–Ҳв–Ҳв–‘в–‘в–‘в–‘в–‘в–‘в–‘в–‘` |

## вңҰ Getting Started

### Prerequisites

```bash
flutter --version  # Requires 3.41.0+
dart --version     # Requires 3.11.0+
```

### Clone & Install

```bash
git clone https://github.com/yourname/link_decay_app.git
cd link_decay_app
flutter pub get
```

### Generate Code

Drift requires code generation:

```bash
dart run build_runner build --delete-conflicting-outputs
```

### Run on Platforms

```bash
# Run on connected Android device/emulator
flutter run

# Run on macOS native desktop
flutter run -d macos

# Run on Windows native desktop
flutter run -d windows
```

### Build Browser Extension (Web)

To build the Chrome / Safari extension bundle:

```bash
flutter build web --web-renderer html --csp
```

Load the compiled `build/web` folder into Chrome as an **unpacked extension**:
1. Open Chrome and navigate to `chrome://extensions/`.
2. Enable **Developer mode** toggle in the top-right.
3. Click **Load unpacked** in the top-left and select the `build/web` directory.

## вңҰ Notification Setup

On first launch, LinkShelf requests notification permission on supported platforms. If granted:
- A daily check runs at **9:00 AM local time**
- If any inbox links are below the freshness threshold, a notification is sent
- The notification survives device reboots via the `BOOT_COMPLETED` receiver (on Android)

To adjust thresholds, go to **Settings вҶ’ Notifications**.
